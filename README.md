# ЗОВ денег

Материалы по брокерской системе и мобильному приложению для торговли ценными бумагами: контракт REST API, документация по курсу/команде и генерация макетов экранов в Figma.

## Структура репозитория

| Путь | Содержимое |
|------|------------|
| **`openapi.yaml`** | Спецификация **OpenAPI 3** REST API приложения: аутентификация JWT, пользователи, ценные бумаги, портфель и связанные эндпоинты. |
| **`docs/`** | Текстовая документация: **`TZ.md`** — техническое задание (архитектура, стек, роли в команде); **`roles.md`** — роли и зоны ответственности. |
| **`zov-figma/`** | Dev-плагин Figma: собирает страницы **«Компоненты»** и **«Экраны»** (макеты **360×800**, Android Compact). Подробные инструкции по сборке и запуску — в [`zov-figma/README.md`](zov-figma/README.md). |

### Каталог `zov-figma/` (кратко)

| Путь | Назначение |
|------|------------|
| **`parts/`** | Исходный код плагина: токены, компоненты, хелперы экранов; подпапка **`screens/`** — отдельные файлы экранов. |
| **`build.mjs`** | Скрипт сборки: склеивает `parts/` в единый скрипт. |
| **`dist/`** | Артефакт **`screens.assembled.js`** — для отладки или вызова Figma MCP (`use_figma`). |
| **`plugin/`** | **`manifest.json`** и сгенерированный **`code.js`** для импорта плагина в Figma (папку `plugin` указывать при импорте). |

Сборка плагина из корня репозитория:

```bash
node zov-figma/build.mjs
```

После правок в `zov-figma/parts/` команду повторяйте. Зависимости: **Node.js**; в Figma нужен шрифт **Inter**.
