# ZOV денег — генерация макетов в Figma

Один раз собирает **страницу «Компоненты»** (библиотека блоков) и **страницу «Экраны»** (ряды макетов **360×800**, Android Compact): вход, главная, **брокерский счёт** (вкладки пополнение / вывод), поиск актива, история, профиль и др.

## Как запустить плагин в Figma

Делать **один раз на компьютере** (пока не смените путь к проекту или `manifest.json`):

1. В терминале из **корня репозитория** (где лежит папка `zov-figma`):

   ```bash
   node zov-figma/build.mjs
   ```

   Так обновляются `plugin/code.js` и `dist/screens.assembled.js`. После правок в `parts/` сборку повторяйте.

2. Откройте **Figma Desktop** (для dev-плагинов так надёжнее) или браузер и нужный **файл дизайна**.

3. Меню **Plugins** → **Development** → **Import plugin from manifest…** (или «Импорт плагина из манифеста…»).

4. Укажите **папку** `zov-figma/plugin` — внутри неё должен лежать `manifest.json`. Figma запомнит плагин.

5. Снова **Plugins** → **Development** → выберите **«ZOV денег · экраны»**. Появится уведомление по завершении; экраны окажутся на страницах «Компоненты» и «Экраны».

При **следующих** запусках шаги 3–4 не нужны — только при первом подключении или смене пути. Шаг 1 — если меняли код в `parts/`.

Плагин при запуске **сам очищает** страницы «Компоненты» и «Экраны» и пересобирает макеты (дубликаты от прошлого запуска не накапливаются).

## Что за папки и файлы

| Путь | Назначение |
|------|------------|
| **`parts/`** | Исходники: правите здесь. Каждый экран — отдельный файл `5x-screen-*.js`. |
| **`build.mjs`** | Склеивает `parts/` в правильном порядке. |
| **`dist/screens.assembled.js`** | **Сгенерированный** единый файл для вставки в **Figma MCP** (`use_figma`) или для отладки. |
| **`plugin/manifest.json`** | Манифест dev-плагина Figma. |
| **`plugin/code.js`** | **Сгенерированный** код плагина (обёртка + тот же скрипт, что в `dist/`). Не править вручную. |

## Порядок фрагментов в `parts/`

Файлы подключаются так (список задан в `build.mjs`, массив `ORDER`):

1. `00-constants.js` — шрифты Inter, размеры, палитра, `txt()`.
2. `20-component-builders.js` — фабрики компонентов (навигация, карточки, строки, шапки).
3. `30-setup-components.js` — страницы «Компоненты»/«Экраны», очистка, выкладка компонентов на полотно.
4. `40-screen-helpers.js` — `androidScreen`, `statusStrip`, `scrollBody`, `appendInstance`.
5. `50-screen-login.js` … `54-screen-profile.js` — сами экраны.
6. `99-return.js` — `return { createdNodeIds, message }` для MCP; при сборке плагина заменяется на `figma.notify`.

Новый экран: добавьте `55-screen-….js`, пропишите имя в `ORDER` в `build.mjs`, снова `node zov-figma/build.mjs`.

## Зависимости

Только **Node.js** для сборки. В Figma должен быть доступен шрифт **Inter** (обычно есть по умолчанию).

## MCP Figma

В лимитах тарифа скопируйте содержимое **`dist/screens.assembled.js`** в вызов инструмента `use_figma` (поле `code`) вместе с `fileKey` вашего файла.

## Если плагин не появляется в списке

Проверьте, что выполнили `node zov-figma/build.mjs` и в `zov-figma/plugin/` есть **`code.js`**. Импорт всегда из папки **`plugin`**, а не из корня `zov-figma`.

## Ошибка `ENOENT ... scripts/figma-plugin-zov/manifest.json`

Figma **запомнила старый путь** к плагину из удалённой папки `scripts/figma-plugin-zov/`.

1. **Figma → Settings (шестерёнка) → Plugins** (или **Manage plugins** в меню плагинов).
2. Вкладка **Development** / **Разработка**: найдите запись, которая вела на старый путь, и **удалите** её (или отключите).
3. Снова **Plugins → Development → Import plugin from manifest…** и укажите **`zov-figma/plugin`** (где сейчас лежит `manifest.json`).

После этого запускайте **«ZOV денег · экраны»** из списка dev-плагинов.

## Сообщения в консоли браузера (можно не трогать)

Часто встречаются и **не мешают** плагину: `fullLoadTime` / `Unexpected arg` (внутренняя Figma), CORS для Gravatar, **404** на превью шрифтов с `127.0.0.1:44950`, предупреждения **Permissions policy** (камера/микрофон в iframe). На работу нашего плагина это обычно не влияет.
